# New CLI Commands (Sprint 41)

Three new commands were added in Sprint 41 to close gaps identified in the Sprint 40 DX usability study.

---

## `lsp done <id>`

Mark a task as complete in `.lsp/tasks.md`.

```bash
lsp done 3          # Mark task #3 as done ([x])
lsp done 3 --dir /path/to/project  # Explicit project path
```

- Searches upward from `process.cwd()` for `.lsp/tasks.md` (works from subdirectories)
- Updates the matching row's Done column from `[ ]` to `[x]`
- Errors with a helpful message if the task ID is not found
- Exits with code 1 if no `.lsp/tasks.md` is found

**Source:** `cli/src/commands/done.ts`

---

## `lsp undone <id>`

Revert a task back to not-done in `.lsp/tasks.md`.

```bash
lsp undone 3        # Mark task #3 as not done ([ ])
```

- Inverse of `lsp done`
- Same path-search logic (upward from `process.cwd()`)

**Source:** `cli/src/commands/done.ts` (exported as `makeDoneCommand(true)`)

---

## `lsp init-backlog`

Convert `.lsp/tasks.md` to `specs/backlog.md` in AutoSpec Sprint 1 format. This is the bridge between LightSpec output and the AutoSpec skill layer (`/execute-ticket`).

```bash
lsp init-backlog              # Create specs/backlog.md from .lsp/tasks.md
lsp init-backlog --yes        # Overwrite existing backlog without prompting
lsp init-backlog --dir /path  # Explicit project path
```

### Time-to-Points Mapping

| Est. Time | Points | Model  |
|-----------|--------|--------|
| 30m       | 1      | haiku  |
| 1h        | 2      | haiku  |
| 2h        | 3      | sonnet |
| 4h        | 5      | sonnet |
| 8h        | 8      | sonnet |
| other     | 3      | sonnet |

### Output format

The generated `specs/backlog.md` follows AutoSpec Sprint 1 format:

```markdown
# ProjectName — SDD Project Backlog

**Version:** 1.0
**Created:** YYYY-MM-DD
**Generated by:** `lsp init-backlog` from .lsp/tasks.md

---

## Sprint 1: Initial Implementation (~X pts)

**Theme:** Implement core features from LightSpec spec
**Status:** 🔲 Planned

| ID | Ticket | Owner | Model | Points | Status | Deps |
|----|--------|-------|-------|--------|--------|------|
| 1.1 | Task title | Fullstack | sonnet | 3 | 🔲 | — |
```

The output is directly compatible with the `/execute-ticket` Claude Code skill.

**Source:** `cli/src/commands/init-backlog.ts`

## Path Robustness (Sprint 41.3)

`lsp status`, `lsp done`, and `lsp graduate` all use an upward directory search for `.lsp/` starting from `process.cwd()`. This means running any of these commands from a subdirectory of your project works correctly without specifying `--dir`.
